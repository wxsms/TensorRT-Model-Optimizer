ModeloptConfig DFlashConfig

Bases: ModeloptBaseConfig

DFlash config for block-wise parallel speculative decoding.

Show default config as JSON
Default config (JSON):

{
   "dflash_block_size": 8,
   "dflash_freeze_base_model": true,
   "dflash_self_logit_distillation": true,
   "dflash_loss_decay_factor": 0.0,
   "dflash_num_anchors": 512,
   "dflash_report_acc": true,
   "dflash_mask_token_id": null,
   "dflash_architecture_config": {},
   "dflash_use_torch_compile": true
}

field dflash_architecture_config: dict

Show details

Config for the DFlash draft module architecture.

field dflash_block_size: int

Show details

Block size for parallel prediction. Draft predicts this many tokens per block.

field dflash_freeze_base_model: bool

Show details

Whether to freeze base model during DFlash module training.

field dflash_loss_decay_factor: float

Show details

Gamma for exponential loss decay weighting (paper Eq.4). Suggested: 7 for block_size=16, 5 for 10, 4 for 8. 0 disables.

field dflash_mask_token_id: int

Show details

Token ID used for masked (unknown) positions. Set explicitly or auto-detected from tokenizer.mask_token_id in main.py.

field dflash_num_anchors: int

Show details

Number of random anchor positions sampled per sequence during training.

field dflash_report_acc: bool

Show details

Whether to report eval accuracy.

field dflash_self_logit_distillation: bool

Show details

Whether to use logit distillation from base model.

field dflash_use_torch_compile: bool

Show details

Whether to use torch.compile on DFlash forward/loss methods.

Eagle config.

Show default config as JSON
Default config (JSON):

{
   "eagle_offline": false,
   "eagle_hidden_state_distillation": false,
   "eagle_self_logit_distillation": true,
   "eagle_freeze_base_model": true,
   "eagle_report_acc": true,
   "eagle_reuse_base_decoder": false,
   "eagle_loss_decay_factor": 0.9,
   "eagle_architecture_config": {},
   "eagle_decoder_type": "llama",
   "eagle_ttt_steps": 3,
   "eagle_mix_hidden_states": false,
   "eagle_use_torch_compile": true,
   "eagle_enable_nvtx": false,
   "eagle_export_rope_scaling": {
      "rope_type": "yarn",
      "factor": 32.0,
      "original_max_position_embeddings": 2048
   },
   "eagle_base_lora": false,
   "eagle_base_lora_rank": 64,
   "eagle_base_lora_alpha": 16.0,
   "eagle_base_lora_target_modules": null,
   "eagle_base_lora_preservation_loss_weight": 0.1,
   "eagle_base_lora_warmup_steps": 0,
   "eagle_base_lora_logits_detach_prob": 0.5
}

field eagle_architecture_config: dict

Show details

The config for eagle module architecture.

field eagle_base_lora: bool

Show details

Whether to add LoRA adapters to the base model for co-training with the EAGLE module. Requires the peft library. Incompatible with eagle_offline=True.

field eagle_base_lora_alpha: float

Show details

LoRA alpha (scaling) for the base model adapters.

field eagle_base_lora_logits_detach_prob: float

Show details

After warmup, probability of detaching base_output_softmax_logits each step. Acts as dropout regularization on the eagle-loss-to-LoRA gradient path through logits, preventing LoRA from degenerating to maximize EAGLE accuracy at the cost of base model quality. 1.0 = always detach (no logits gradient), 0.0 = never detach.

field eagle_base_lora_preservation_loss_weight: float

Show details

Weight for the preservation loss that minimizes the KL divergence between the LoRA-adapted base model output and the original base model output.

field eagle_base_lora_rank: int

Show details

LoRA rank for the base model adapters.

field eagle_base_lora_target_modules: list | None

Show details

List of module name patterns to apply LoRA to in the base model (e.g. [‘q_proj’, ‘v_proj’]). None uses peft defaults.

field eagle_base_lora_warmup_steps: int

Show details

Number of warmup steps where LoRA is frozen and only the EAGLE draft head trains. After warmup, LoRA is enabled for co-training.

field eagle_decoder_type: str

Show details

The class of eagle decoder to use. Available options: llama, kimik2

field eagle_enable_nvtx: bool

Show details

Whether to enable NVTX ranges for profiling eagle forward/loss methods.

field eagle_export_rope_scaling: dict

Show details

The rope_scaling config to inject into the exported HuggingFace model config. Applied when the training rope_type is ‘default’ (no scaling). Set to empty dict {} to disable rope scaling injection at export.

field eagle_freeze_base_model: bool

Show details

Whether to freeze base model during eagle module training.

field eagle_hidden_state_distillation: bool

Show details

Whether to use feature hidden states distillation.

field eagle_loss_decay_factor: float

Show details

The decay factor for multiple eagle_loss.

field eagle_mix_hidden_states: bool

Show details

Whether to mix hidden states of multiple TTT steps. It is a technique to reduce training cost.

field eagle_offline: bool

Show details

Whether to use detached Eagle.

field eagle_report_acc: bool

Show details

Whether to report eval accuracy.

field eagle_reuse_base_decoder: bool

Show details

Whether to reuse base model decoder in eagle module.

field eagle_self_logit_distillation: bool

Show details

Whether to use logit distillation.

field eagle_ttt_steps: int

Show details

The number of train-time-test steps in training.

field eagle_use_torch_compile: bool

Show details

Whether to use torch.compile on eagle forward/loss methods for faster training.
